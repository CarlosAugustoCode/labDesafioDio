O que é o HTML?

O HTML é uma linguagem de marcação utilizada na construção de páginas na Web. Documentos HTML podem ser interpretados por navegadores. Conforme já citado, o HTML não é uma linguagem de programação e sim, uma linguagem de marcação de hipertexto.

A tecnologia é fruto da junção entre os padrões HyTime e SGML. HyTime é um padrão para a representação estruturada de hipermídia e conteúdo baseado em tempo.

Fonte: Wikipedia.

O que são Clients?

Reforcei na aula que o conceito de Client, na mais é que o dispositivo que usuário utiliza para interagir com um servidor para troca de dados, através de um navegador, aplicativo, software, por exemplo.

Um cliente é um programa que acessa um serviço que é disponibilizado por um servidor. O cliente faz uma solicitação para um serviço e o servidor executa esse serviço. Um exemplo de cliente são os browsers web. Os programas do cliente geralmente lidam com interações do usuário e geralmente solicitam dados ou solicitam alguma modificação de dados em nome de um usuário.

Fonte: Techenter.

O que são Servers.

Servers são unidades computacionais com grande capacidade de armazenamento e de processamento onde o objetivo é conectar outros computadores através de uma rede.

Um servidor é um programa de computador funcionando para atender demandas de outros programas, os clientes. Exemplos de servidores incluem servidores da web, servidores de e-mail e servidores de arquivos, servidores de banco de dados, entre outros.

Fonte: Techenter.

Estrutura Básica do HTML.

Para criação de páginas na internet, o desenvolvedor precisa dominar e entender a estrutura do HTML, compreendendo as sintaxes e criação dos elementos e tags necessárias na programação. Com isso, consegui separar as mais importantes da estruturação

  1. <.!DOCTYPE html>
    • Esta tag precisa estar presente no inicio de todos os arquivos HTML, nela está presente também a tag "body", que indica aos nagevadores todas as configurações do arquivo HTML.

  2. <.HTML>
    • Essa tag representa o início dos elementos de uma página. Todos os demais elementos precisam ser descendentes dessa tag, seja diretamente ou indiretamente.

  3. <.HEAD>
    • A tag de cabeçalho (Head) contém o conteúdo que é visualizado pela máquina, tal como os scripts, links externos e metadados. Não é um elemento visual.

  4. <.TITLE>
    • Serve para descrever o título da página presente na aba atual do navegador.

  5. <.BODY>
    • É uma das tags mais populares e importantes do HTML, dentro dela adicionamos os conteúdos que podem ser visualizados pelo usuário final. É uma tag única e pode ser considerada a tag pai de todos os elementos de conteúdo.

Listas Ordenadas e Não-ordenadas.

A principal diferença entre a lista ordenada e a lista não-ordenada está na maneira com que os itens de cada lista são apresentados - a lista ordenada enumera os elementos; a lista não-ordenada coloca marcadores antes de cada elemento.

  1. Lista Ordenada
    • Uma lista ordenada é delimitada pelas etiquetas OL e /OL (Ordered List). Cada item da lista é delimitado pelas etiquetas LI e /LI (List Item).


  2. Lista Não-ordenada
    • A estrutura da lista não-ordenada é semelhante a da lista ordenada, porém, ao invés de ser construída com o elemento OL, é construída com o elemento UL (Unordered List).
